MPatch Preferences page

The framework for MPatches is modular and can be extended with several extensions. Since most extensions are required, default implementations for them are already shipped. The preferences dialog defines the default choice for each extension point (in fact, the framework throws warnings if for some extension points there is no default extension selected).

Please open Window -> Preferences -> EMF Compare -> MPatch:

  • Several transformations in the MPatchcreation process can be performed. The default selection and order is defined here.
  • Symbolic references are used to replace the direct model references.
  • Model descriptors are used to store sub-models in the differences, for example added or removed model elements.
  • The strategy for symbolic reference resolution during the application of differences can be chosen here.
  • The difference application engine can be chosen here.

The important selections for normal use are the first two options. Please see the transformation page for details. The (outdated) design document contains further technical details about the design and how to add extensions.
